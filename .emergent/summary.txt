**Code Architecture**


**Key Technical Concepts**
- **Merchant Pool System:** Manages temporary cryptocurrency addresses for receiving payments on behalf of merchants.
- **Per-Chain Sweep Logic:** A configurable system that defines how and when admin fees are collected from pool addresses, with different strategies for different types of blockchains.
- **UTXO Batch Transfer:** A cost-saving technique for Bitcoin-like chains where merchant payout and admin fee collection happen in a single transaction with multiple outputs.
- **Supervisor:** A process control system used to manage backend/frontend services and run the dependency installation script on boot.
- **Sequelize:** A Node.js ORM used for interacting with the PostgreSQL database.

**key DB schema**
- **tbl_merchant_temp_address:**  - Tracks temporary payment addresses.
- **tbl_merchant_wallet:**  - Stores the merchant's master HD wallets.
- **tbl_customer_transaction:**  - Records all processed transactions, with a unique constraint on  to prevent duplicates.
